About tail.writer

A light-weight, powerful and Open Source GitHub Flavored Markdown editor, written in pure vanilla JavaScript with a jQuery and a MooTools implementation.

Documentation

This documentation is still Work in Progress and doesn't contain the complete tail.writer environment!

Install & Embed

You can download and install the tail.writer package using one of the 2 buttons (.tar or .zip) above in the header, or by callin NPM npm install tail.writer or Yarn yarn add tail.writer directly in your console, if you have the respective package manager installed of course.

You can also use the CDN jsDelivr or unpkg to receive the tail.writer script without to download the package itself: 

https://cdn.jsdelivr.net/npm/tail.writer@latest/
https://unpkg.com/tail.writer/

No matter which way you take, you still need to embed the JavaScript as well as the Stylesheet in your HTML document and please don't forget to embed the respective theme-stylesheet and the desired language JavaScript file too. We recommend to load the stylesheets within the HTML header and the JavaScript files directly after the opened <body> HTML tag. 

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8" />
        <title>Example Site</title>

        <link type="text/css" rel="stylesheet" href="css/tail.writer.css" />
        <link type="text/css" rel="stylesheet" href="css/tail.writer.github.css" />
    </head>
    <body>
        <script type="text/javascript" src="js/tail.writer.min.js"></script>
        <script type="text/javascript" src="langs/tail.writer-de.js"></script>
    </body>
</html>

How to Use

The tail.writer script needs a textarea element, where it builds up the respective environment. Just add this HTML Element anywhere you need and call the tail.writer() function, optimally within the DOMContentLoaded event to prevent beforeLoading failures. The global tail.writer() function allows 2 arguments. The first MUST be

  • ... a (HTML) Element Object (getElementById, querySelector)
  • ... a (HTML) Element List Object (getElementsBy*, querySelectorAll)
  • ... an Array with Element Objects ([getElementById, getElementById])

or just a STRING selector like "textarea", which equals the querySelectorAll method.

The second parameter CAN be an object with respective key => value based options, check out the list below to find all available options. You can also skip this and just use the default settings. 

document.addEventListener("DOMContentLoaded", function(){
    tail.writer("textarea");
});

How to Configurate

The tail.writer Markup Editor should now already work, but you can configure the environment with the following options: 

document.addEventListener("DOMContentLoaded", function(){

    // The settings below are the defaults!
    tail.writer("textarea", {
        width: "100%",
        height: ["200px", "500px"],
        classes: "",
        resize: true,
        indentTab: false,
        indentSize: 4,
        toolbar: [
            "headers", "|", "bold", "italic", "strikethrough", "|", "quote", "code",
            "codeblock", "indent", "outdent", "|", "link", "image", "table", "hr", "|",
            "list:unordered", "list:ordered", "|", "preview"
        ],
        toolbarPosition: "top",
        tooltip: "top",
        statusbar: true,
        animate: true
    });
});

Available Options

width
string
This is the respective width, which the tail.writer Interface should be fill up.
height
array
The first parameter configures the minHeight of the tail.writer editor, the second parameter the maxHeight, which depends on the resize option!
classes
string
This option adds additional class names to the main tail.writer object.
resize
bool
Set this option to true to allow the editor to resize itself (until the maxHeight value of the height option has been reached), use false to keep the editor to the minHeight value only!
indentTab
bool
Set this variable to true, to use Tabs \t instead of Spaces for indenting, and use false to use spaces, which number is defined in indentSize.
indentSize
integer
This option controls the number of spaces for indenting, when indentTab is set to false.
toolbar
array
This option builds up the markup action toolbar, fill it with the action described below to build your own action toolbar.
toolbarPosition
string
This option should change the position of the toolbar, but it currently does just nothing.
tooltip
string
This option should change the position of the markup action tooltips, but currently can only disable the tooltips at all by using false as value.
statusbar
bool
Set this option to true to enable the statusbar, which contains Line / Word / Character counter or use false to hide the statusbar.
animate
bool
Set this option to true to enable the tooltip and dialog / dropdown animation, use false to disable all animations.

Available Actions

Some available actions allows one or more parameters to configure it's behavior (or the markup syntax, for example). The action syntax is build up like: action_name:param1,param2, each action can also be called without any parameter set.

header:size
A single header (<h1> up to <h6>), the first parameter configures the header type (1 up to 6).
headers:type
A dropdown field with 6 (or if the type parameter is set to "x3" with just 3) headers.
bold
The default bold action, which used the **bold** syntax.
italic
The default italic action, which used the _italic_ syntax.
underline
The Markdown Markup doesn't support underlined text, so this action uses the default HTML <u>underlined</u> tags.
strikethrough
The default strikethough action, which used the ~strikethough~ syntax.
code
The inline code action, which used the `code` syntax.
hr
The horizontal rule action, which uses "----------" (10 x "-") as markup syntax.
code
The block code aka <pre> action, which uses "```\n\n```" as markup syntax.
quote
The quote action, which uses ">\t" as markup syntax.
list:type
The default list action, which can be "unordered", "ordered", "unchecked" or "checked" depending on the passed parameter.
link:type
The default link action, which inserts the syntax directly of no parameter is passed. Use the "dialog" or "dropdown" option to open respectivly.
image:type
The default link-image action, which inserts the syntax directly of no parameter is passed. Use the "dialog" or "dropdown" option to open respectivly.
table:type
The table action can be opened as "dialog" or as "dropdown", depending on the passed parameter.
indent
This option just adds an indention "\t" (or the configured number of spaces) on the respective position(s).
outdent
This option just removed an indention "\t" (or the configured number of spaces) on the respective position(s).
preview
This option renders the respective Markup text, and is only available if an supported JavaScript parser is available!
